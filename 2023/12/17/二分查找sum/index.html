基于已排好序的数据

c++ 在STL 中有多个二分查找函数

  • binary_search：查找某个元素是否出现。
    • binary_search(begin,end,index);
    • 时间复杂度 log2(N)+2
  • lower_bound() 查找第一个大于或等于某个元素的位置
  • upper_bound() 用法同于lower_bound();没有等于
    • lower_bound(begin,end,x,[geaster< int >()])
    • 添加【geaster < int >()】参数后将重新设置查找规则改为【查找第一个小于等于x的下标值】
    • 如果所有的元素都 小于x|大于x 则返回end（end是越界的）
    • 返回的值为迭代器 如果要得到下标值 则用返回值-begin()
    • 时间复杂度 O (logN)

mid = (l+r+1)/2;和mid = (l+r)/2;二分查找中有什么区别

Gemini

在二分查找算法中，mid = (l+r+1)/2和mid = (l+r)/2的区别在于，前者将中间索引向上取整，而后者将中间索引向下取整。

当二分查找算法应用于查找一个元素在有序数组中的索引时，向上取整和向下取整都会产生不同的结果。

向上取整（mid = (l+r+1)/2）

向上取整将中间索引向上取整为最接近中间值的整数。如果中间索引是一个整数，则向上取整后仍是同一个整数。如果中间索引是一个小数，则向上取整后将变为下一个整数。﻿input﻿

例如，如果l = 0、r = 9，则mid = (0+9+1)/2 = 5。

向下取整（mid = (l+r)/2）

向下取整将中间索引向下取整为小于或等于中间值的最大的整数。如果中间索引是一个整数，则向下取整后仍是同一个整数。如果中间索引是一个小数，则向下取整后将变为上一个整数。

例如，如果l = 0、r = 9，则mid = (0+9)/2 = 4。

向上取整和向下取整的区别在于，当中间索引是一个小数时，向上取整的结果比向下取整的结果大1。

在二分查找算法中，向上取整和向下取整的区别会影响算法的性能。对于某些特定的数据结构，向上取整或向下取整可能会更有效。一般来说，向上取整会产生更小的错误，但代价是牺牲了一定程度的性能。
